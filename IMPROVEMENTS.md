# Эволюция функциональности Telegram-бота AI Lawyer

## Введение

В процессе разработки Telegram-бота AI Lawyer было реализовано две
основные версии продукта.\
Первая версия была направлена на проверку базовых продуктовых гипотез и
ценности решения для пользователей.\
После проведения пользовательского тестирования функциональность
продукта была расширена и доработана с учетом полученной обратной связи.

Ниже приведено сравнение функциональности первой версии продукта и
финальной версии после тестирования.

------------------------------------------------------------------------

# Первая версия продукта

Первая версия бота была ориентирована на быстрый первичный анализ
договора.\
Основная задача заключалась в том, чтобы проверить, помогает ли
автоматический анализ документа пользователям быстрее понимать
содержание договора и выявлять риски.

В первой версии пользователю были доступны следующие функции.

## 1. Загрузка договора

Пользователь мог загрузить договор в формате .doc или .docx
непосредственно в Telegram-бот.

После загрузки:

-   текст документа извлекался из файла
-   передавался в языковую модель для анализа
-   модель формировала структурированное описание договора

Таким образом пользователь мог получить результат анализа без
необходимости вручную читать весь документ.

------------------------------------------------------------------------

## 2. Отрисовка простой одноуровневой схемы

После обработки договора система строила простую одноуровневую
BPMN-схему, которая отображала общую структуру договора.

Схема показывала:

-   основные разделы или этапы договора
-   последовательность логики документа

Цель этой визуализации --- помочь пользователю быстро понять структуру
договора и ориентироваться в нем.

------------------------------------------------------------------------

## 3. Моментальная выдача краткого описания и ключевых рисков

Одновременно со схемой пользователю автоматически показывались:

-   краткое описание содержания договора
-   перечень ключевых рисков

Таким образом пользователь сразу получал три вида информации:

-   визуальную структуру договора
-   общее описание документа
-   основные риски

Основной пользовательский сценарий выглядел следующим образом:

пользователь загружает договор → получает схему → читает краткое
описание → изучает риски → принимает решение о дальнейшем чтении или
согласовании документа

------------------------------------------------------------------------

# Результаты пользовательского тестирования

Тестирование первой версии продукта показало, что:

-   краткое описание действительно помогает быстро понять содержание
    договора
-   блок с рисками является наиболее востребованной частью анализа
-   визуальная схема помогает ориентироваться в документе

Однако пользователи также выявили ряд ограничений первой версии.

Основные проблемы:

-   схема была слишком поверхностной
-   не было понятно кто отвечает за конкретные действия
-   отсутствовали инструменты для дальнейшей работы с договором внутри
    продукта

Это позволило сформулировать направления развития продукта.

------------------------------------------------------------------------

# Финальная версия продукта (после тестирования)

После анализа пользовательской обратной связи функциональность бота была
существенно расширена.

## 1. Более детальная структуризация договора

Во второй версии схема стала более детализированной.

Теперь система выделяет:

-   конкретные действия в рамках договора
-   последовательность выполнения этих действий
-   стороны, ответственные за каждое действие

Это позволило перейти от простой структуры документа к модели реального
процесса выполнения договора.

------------------------------------------------------------------------

## 2. Добавление интерактивного меню действий

Во второй версии под результатом анализа появились кнопки для работы с
договором.

Пользователь может выбрать конкретное действие:

-   Краткое описание договора
-   Ключевые риски
-   Новый договор
-   Внести правки

Такой интерфейс позволяет пользователю запрашивать только нужную
информацию и работать с договором более гибко.

------------------------------------------------------------------------

## 3. Возможность внесения правок прямо в боте

Одним из ключевых улучшений стала возможность вносить правки в договор
прямо внутри Telegram-бота.

Теперь пользователь может:

-   указать изменения в формулировках
-   уточнить отдельные пункты
-   изменить условия договора

После этого система повторно анализирует текст и генерирует обновленную
BPMN-схему, отражающую внесенные изменения.

Это превращает продукт из инструмента анализа в инструмент интерактивной
работы с договором.

------------------------------------------------------------------------

# Итог

Развитие продукта прошло путь от инструмента быстрого анализа договора к
интерактивному помощнику для работы с договором.

Основные изменения между версиями:

  Первая версия                           Финальная версия
  --------------------------------------- -------------------------------------
  Простая одноуровневая схема             Детализированная схема с действиями
  Общая структура договора                Привязка действий к сторонам
  Моментальная выдача описания и рисков   Интерактивное меню действий
  Только анализ                           Возможность внесения правок
  Статичный результат                     Обновление схемы после изменений

Таким образом, тестирование с пользователями позволило значительно
улучшить продукт и приблизить его к реальным рабочим сценариям
использования.
